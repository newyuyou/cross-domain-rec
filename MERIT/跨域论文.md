# MERIT双塔架构重构总结

## 1. 项目背景与目标

### 1.1 原MERIT架构问题
- **三塔设计**：混合塔h_m（全序列）、域A塔h_a、域B塔h_b
- **限制**：所有Cross-Attention都有因果mask（is_causal=True），跨域信息受限
- **推理不一致**：训练时跨域信息受时间戳限制，推理时取最后一个位置

### 1.2 重构目标
改为**每个域双塔架构**（纯净塔+混合塔）：
- **纯净塔**：严格因果，仅看本域历史
- **混合塔**：Self-Attention因果 + Cross-Attention**无mask**（可获取跨域全量信息）

这样允许混合塔在Cross-Attention中获取跨域的最新行为信息，不受时间戳限制。

---

## 2. 架构对比

| 特性 | 原MERIT | 新MERITDual |
|------|---------|-------------|
| 塔结构 | 3塔（m, a, b） | 2双塔（A: pure+mixed, B: pure+mixed） |
| Cross-Attention | 全部带因果mask | 混合塔**无mask**，可看全序列 |
| 信息利用 | 受时间戳限制 | 跨域信息充分利用 |
| 预测表示 | h_a + h_b + h_m | (h_pure_a + h_mixed_a) 或 (h_pure_b + h_mixed_b) |

## 3. 关键修改文件

### 3.1 models/attention.py（新增类）
新增 `CrossAttentionNoMask` 类：
- 无因果mask的Cross-Attention
- 用于混合塔获取其他域的全量信息（包括"未来"位置）

### 3.2 models/DualDecoder.py（新建）
核心实现，包含两个类：

**DomainDualTower**：
- `forward_pure()`：纯净塔，因果Self-Attn + FFN
- `forward_mixed()`：混合塔，因果Self-Attn + 无mask Cross-Attn + FFN

**MERITDual**：
- 两个DomainDualTower（tower_a, tower_b）
- 两阶段前向：先各自过pure塔，再交叉进行mixed塔
- 融合策略：默认相加融合 h_pure + h_mixed
- 保持与原模型相同的损失计算和评估方式

### 3.3 trainer_dual.py（新建）
- `TrainerDual`类适配新架构
- 简化的损失计算（去掉mixed序列损失）
- 使用fused表示进行预测

### 3.4 main_dual.py（新建）
- 新架构的入口文件
- 与原main.py参数完全一致

---

## 4. 信息流动机制

```
Domain A序列 → Pure Tower A → h_pure_a ──┐
                                         ├──→ Mixed Tower A → h_mixed_a
Domain B序列 → Pure Tower B → h_pure_b ──┘
                                         （Cross-Attn无mask，可看全序列）

预测Domain A：h_pred_a = h_pure_a + h_mixed_a
预测Domain B：h_pred_b = h_pure_b + h_mixed_b
```

**关键设计**：混合塔A的Cross-Attention使用B的纯净塔输出作为KV，且**无因果mask限制**，因此可以看到B域的全序列信息。

## 8. 关键实现细节

### 8.1 Mask语义（与原MERIT一致）
- `mask_a`：Domain A物品（seq_m > n_item_a）
- `mask_b`：Domain B物品（0 < seq_m <= n_item_a）

### 8.2 两阶段前向（解决循环依赖）
```python
# Step 1: 各自过pure塔
h_pure_a = tower_a.forward_pure(h_a, mask_a)
h_pure_b = tower_b.forward_pure(h_b, mask_b)

# Step 2: 交叉进行mixed塔（用对方的pure输出）
h_mixed_a = tower_a.forward_mixed(h_a, mask_a, h_pure_b)
h_mixed_b = tower_b.forward_mixed(h_b, mask_b, h_pure_a)
```

### 8.3 融合策略（可扩展）
- 当前：简单相加 `h_pure + h_mixed`
- 可选：可学习门控（代码中已注释，需要时开启）

## 9. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 过度依赖跨域未来信息 | 残差连接保持self-attn主导地位 |
| 训练不稳定 | LayerNorm + Dropout正则化 |
| 信息泄露误解 | 设计文档清晰说明意图 |
